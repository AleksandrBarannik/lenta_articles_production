Архитектура FutureSlice(FSD):
1)Layers:
        app:
        настройки, стили и поставщики для всего приложения.

        pages:
        композиционный слой для создания полных страниц
        из объектов, функций и виджетов.


        widgets:
        композиционный слой для объединения сущностей и функций в значимые блоки.
        (например, IssuesList, UserProfile)


        features:
        взаимодействие с пользователем,
        действия, которые приносят пользу пользователю для бизнеса.
        (например, SendComment, AddToCart, UsersSearch)


        entities:
        бизнес-объекты.
        (например, пользователь, продукт, заказ)

        shared:
        функциональность многократного использования,
        отделенная от специфики проекта / бизнеса.
        (например, UIKit, библиотеки, API)

2. Slices:
        Затем есть фрагменты, которые разделяют код по бизнес-областям.
        Это упрощает навигацию по вашей кодовой базе,
        поскольку логически связанные модули расположены близко друг к другу.

3. Segments:
        Каждый фрагмент, в свою очередь, состоит из сегментов.
        Это крошечные модули, которые предназначены для разделения кода
        внутри фрагмента по его техническому назначению.
        Наиболее распространенными являются ui, model , api и lib
